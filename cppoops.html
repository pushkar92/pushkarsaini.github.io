OBJECT BASED PROGRAMMING

It is subset of object oriented programming as it implements some features of object oriented programming - abstraction , information hiding , function overloading , classes etc. But it does not implement inheritance and polymorphism.

It suffers from one major limitation that is it is not able to represent real world relationships that exists among objects.

 

OBJECT ORIENTED PROGRAMMING(OOPS)

It follows four properties : abstraction , encalsulation , inheritance and polymorphism.It is superset of object based programming , it overcomes its limitations by representing real world relationships that exists among objects with the help of inheritance and polymorphism.

Let's understand concepts of oops :

1. ENCAPSULATION

.Encapsulation means combining the data and the functions that operate on that data under a single unit.

These functions are called as member functions or methods in C++.

Remeber that data can't be accessed directly because data is hidden, so as to protect it from accidental changes.

2. ABSTRACTION

Abstraction means to represent only essential features and hiding the background details.

It means show only that much information as it requires for the client to know.

Eg: when mechanic desgins car he doesn't tell how the car is made , how breaks are made , how engine is designed etc he only tell the driver how to drive i.e gear and steer handling.

3. INHERITANCE

Inheritance means ability of one class to derive the properties of another class.

4. POLYMORPHISM

Polymorphism is the ability for a data to be processed in more than one form.

5. MODULARITY

Modularity is called as the act of partitioning the program into individual components.

The justification for partitioning the program is that :

=> it reduces complexity of program to some degree.

=> it creates a number of well-defined , documented boundaries within the program.
