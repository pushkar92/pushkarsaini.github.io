OBJECT ORIENTED PROGRAMMING


CONCEPTS OF OBJECT ORIENTED PROGRAMMING

 

1.  DATA ABSTRACTION : Abstraction is an act of representing only essential features without including the background details. eg: you are driving a car , you only need to know how to drive it and you don't care about its internal working how car is made (wiring , motor working).

 

2. DATA ENCAPSULATION : it is the way of combining both data and the functions that operate on data under a single unit.

3.  INHERITANCE : is the capablitiy of one class of things to inherit the characteristics from another class.

4. POLYMORPHISM : is the ability for a message or data to be processed in more than one form.

 

 

Three components

CLASS

A class is a blueprint or template from which objects are made.

or

A class is a way to bind the data describing an entity and its associated functions together.

OBJECT

object is an entity that make use of the class.

an object has properties(data) and the behaviour(methods/functions).


REFERENCE :-

reference variable is used to refer to an object .


A class consist of three parts :-

1.  Fields

          1.1 Fields are of two types:

            -> variables

            -> constants - constants are declared using keyword final.

                    eg:- final double pi = 3.14152967

2. Methods

2.1 Methods are of three types :-

Generalized methods

      a. Concrete methods

      b. Abstract methods

      c. Final methods

2. Specialized Methods

          a. Constructor

           b. Finalizer

                              

3. Static objects

 


WHAT IS CONSTRUCTOR ?

Special methods having some features

1. Same name as class name

2. No return type

3. Used to initialize fields of an object

4. Can be overloaded

5. We can have private constructor

6. If no constructor is created then default constructor is created automatically but if we create any parameterized constructor then default constructor is not created automatically

 

TYPE OF CLASS MEMBERS

- The class members can be of two types

o Static or Class Members

o Non-static or Instance Members (default)

- Static members do not need an object to call them while non-static members

always need an object to call them

- Use static keyword to declare the static members
